# 常见性能问题及解决方案
在前端开发过程中，性能问题是常见的挑战之一。以下是一些常见的性能问题及其解决方案：
## 资源压缩
聚焦代码、图片、静态资源三类核心压缩方式，从开发 / 打包阶段落地，兼顾体积与加载速度：
- 代码压缩：通过 Webpack/Vite 自带的 TerserPlugin 压缩 JS 代码（删除注释、空格、变量名混淆），压缩 CSS（合并样式、去除无用代码），开发中遵循按需引入（如 Element Plus/Vant 组件库只引入使用的组件，而非全局引入），减少打包体积。
- 图片压缩：使用工具如ImageOptim、TinyPNG等对图片进行压缩，减少文件大小，同时保持图片质量。
- 静态资源压缩：使用工具如Gzip、Brotli等对CSS、JavaScript等静态资源进行压缩，减少传输数据量，提高加载速度。
## 懒加载
核心原则 **[按需加载，非首屏 / 非可视区资源不加载]，避免首屏一次性加载所有资源导致白屏，主要应用于路由、图片、组件**：

- 路由懒加载：使用动态导入（import()）实现路由组件的懒加载，只有在用户访问该路由时才加载对应的组件。
示例：
```javascript
const Home = () => import('@/views/Home.vue')
```
- 图片/视频懒加载：给 img 标签添加loading="lazy"原生属性（浏览器原生支持），或通过 JS 监听滚动事件，判断元素是否进入可视区，再设置 src 属性；配合占位符（骨架屏 / 纯色占位）提升用户体验。
使用Intersection Observer API或第三方库（如lazysizes）实现图片的懒加载，只有当图片进入可视区域时才加载。
- 组件懒加载：对非关键组件使用懒加载，减少首屏渲染时的资源消耗。对页面中非首屏渲染的组件（如弹窗、底部导航、详情页子组件），通过 Vue 的defineAsyncComponent实现异步加载，避免页面初始化时渲染所有组件。
## 缓存策略
利用浏览器缓存、HTTP 缓存、本地存储，让重复访问的资源无需重新从服务器请求，核心分强缓存和协商缓存（HTTP 缓存）+ 前端本地缓存：
- 强缓存：通过 HTTP 响应头Cache-Control（优先级更高）和Expires设置，指定资源缓存有效期，有效期内浏览器直接从本地缓存读取，不向服务器发请求。
常用配置：Cache-Control: max-age=86400（资源缓存 24 小时），适用于**不常更新的静态资源**（如图片、字体、打包后的 JS/CSS）。
- 协商缓存：强缓存失效后触发，浏览器携带资源标识（Last-Modified/If-Modified-Since 或 ETag/If-None-Match）向服务器发请求，服务器判断资源是否更新：未更新则返回 304 状态码，浏览器使用本地缓存；已更新则返回 200 和新资源。适用于**偶尔更新的静态资源**。
- 前端本地缓存：对高频访问的非敏感数据（如商品列表、用户基本信息、接口返回的静态数据），通过localStorage（永久存储，手动清除）/sessionStorage（会话存储，页面关闭失效）/Vuex/Pinia + 持久化插件缓存，避免每次页面刷新都重新请求接口；对**高频变动的临时数据**（如购物车临时数据）使用sessionStorage，防止数据丢失。


## 1. 页面加载速度慢
### 解决方案：
- 采用懒加载技术，延迟加载非关键资源。
- 使用CDN加速静态资源的加载。
- 进行资源压缩和合并，减少HTTP请求次数。
## 2. JavaScript执行效率低
### 解决方案：
- 避免在关键路径上使用复杂的计算，尽量将其放在Web Worker中执行。
- 使用节流和防抖技术，减少不必要的函数调用。

## 3. 图片加载慢
### 解决方案：
- 使用适当格式的图片（如WebP）并进行压缩。
- 采用懒加载技术，延迟加载非可视区域的图片。

## 4. CSS渲染阻塞
### 解决方案：
- 将关键CSS内联到HTML中，减少外部请求。
- 使用媒体查询，按需加载不同的CSS文件。

## 5. 网络请求延迟
### 解决方案：
- 使用Service Worker缓存静态资源，减少网络请求。
- 合并API请求，减少请求次数。
## 6. 内存泄漏
### 解决方案：
- 使用Chrome DevTools等工具检测内存泄漏，及时释放不再使用的对象。
- 避免使用全局变量，尽量将变量限制在函数内部。
## 7. DOM操作频繁
### 解决方案：
- 使用文档片段（DocumentFragment）批量操作DOM，减少重排和重绘。
- 合并多个DOM操作为一次操作，减少DOM访问次数。
## 8. 过度使用动画和特效
### 解决方案：
- 使用CSS动画代替JavaScript动画，利用GPU加速。
- 限制动画的数量和复杂度，避免影响页面性能。
## 9. 不合理的资源加载顺序
### 解决方案：
- 使用`<link rel="preload">`提前加载关键资源。
- 按需加载非关键资源，使用`async`和`defer`属性优化脚本加载顺序。
## 10. 不合理的组件设计
### 解决方案：
- 采用懒加载和按需加载的策略，减少初始加载时的资源消耗。
- 进行组件拆分和优化，避免过度嵌套和复杂的组件结构。
通过识别和解决这些常见的性能问题，可以显著提升前端应用的性能和用户体验。
